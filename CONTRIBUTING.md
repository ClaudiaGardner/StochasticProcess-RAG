# Contributing to StochasticProcess-RAG

感谢你对本项目的关注！我们欢迎任何形式的贡献。

## 如何贡献

### 报告 Bug

如果你发现了 bug，请：

1. 检查 [Issues](https://github.com/ClaudiaGardner/StochasticProcess-RAG/issues) 确认问题是否已被报告
2. 如果没有，创建一个新的 Issue，包含：
   - 清晰的标题
   - 详细的问题描述
   - 复现步骤
   - 预期行为和实际行为
   - 你的环境信息（Python 版本、操作系统等）

### 提出新功能

如果你有好的想法：

1. 先在 [Discussions](https://github.com/ClaudiaGardner/StochasticProcess-RAG/discussions) 讨论
2. 获得反馈后，创建一个 Feature Request Issue
3. 详细描述功能的用途和实现思路

### 提交代码

1. **Fork 项目**
   ```bash
   # 点击 GitHub 页面右上角的 Fork 按钮
   ```

2. **克隆你的 Fork**
   ```bash
   git clone https://github.com/你的用户名/StochasticProcess-RAG.git
   cd StochasticProcess-RAG
   ```

3. **创建特性分支**
   ```bash
   git checkout -b feature/amazing-feature
   ```

4. **进行修改**
   - 遵循现有的代码风格
   - 添加必要的注释
   - 确保代码能够正常运行

5. **提交更改**
   ```bash
   git add .
   git commit -m "Add: 简短描述你的更改"
   ```

   提交信息格式：
   - `Add: 新增功能`
   - `Fix: 修复问题`
   - `Update: 更新内容`
   - `Refactor: 重构代码`
   - `Docs: 文档更新`

6. **推送到你的 Fork**
   ```bash
   git push origin feature/amazing-feature
   ```

7. **创建 Pull Request**
   - 在 GitHub 上打开你的 Fork
   - 点击 "New Pull Request"
   - 填写 PR 描述，说明你的更改

## 代码规范

- 使用 Python 3.8+ 语法
- 遵循 PEP 8 代码风格
- 函数和类添加文档字符串
- 复杂逻辑添加注释

## 测试

在提交 PR 前，请确保：

- [ ] 代码能够正常运行
- [ ] 没有引入新的错误
- [ ] 更新了相关文档

## 文档

如果你的更改影响了用户使用方式：

- 更新 README.md
- 更新 README_CN.md
- 必要时添加示例

## 行为准则

- 尊重所有贡献者
- 保持友好和专业
- 接受建设性的批评
- 关注项目的最佳利益

## 问题？

如有任何疑问，欢迎在 [Discussions](https://github.com/ClaudiaGardner/StochasticProcess-RAG/discussions) 提问。

感谢你的贡献！🎉
